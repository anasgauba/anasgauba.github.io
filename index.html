Agent-Based Model for Covid-19 Screening

This project extended Covid-19 Spread and Evolution project. In this project, we explored what happens if we keep the economy open while we increase the percentage of people being tested and then quarantine them. Will this flatten the curve? We modeled this with the help of an Agent-Based Model (ABM) using SIRQ dynamics. The simulation above shows several communities of people on the outskirts of the city while other buildings such as restaurants, groceries, etc. in the center part of the city. People from different communities carry out their daily tasks, keeping the economy open while more testing is in place. As disease spreads, more people get tested and then get quarantined (shown in while dots).

Covid-19 Spread and Evolution

Covid-19 Spread and Evolution project was part of Complex Adaptive Systems class at UNM. As we all know, Covid-19 broke out in United States around February 2020, making so many people sick. In this project an investigation was conducted to study how different variants of viruses attach to humans and mutate, hence, evolving over time. This project also conducted study to understand how a variant of the disease can spread in the population living in a quarantined city. To achieve this, Cellular Automata (CA) with SIR dynamics was used. As part of this study, another study was also conducted to see what happens when a second variant of the disease is introduced in the population and how it can evolve to coexist with the first variant. This was achieved with the help of a Genetic Algorithm.

Cretaceous Gardens Controller

Cretaceous Gardens Controller (CGC) simulation is inspired from Jurassic Park. The CGC is a system that helps control the entire software application for a theme park that has a T-Rex exhibit. It is a centralized system that manages safety protocol for employees and visitors to the park, emergency protocol for the park, and handles sales and tracking of devices as well as communication. It's a capstone project from Software Engineering course taken at UNM. This capstone project included writing several technical documents, design, and an implementation.

Auction House

Auction House is a distributed system based project. There are three main components in this program which are Agent, Bank, and Auction House. Each main component uses proxy communication to communicate with other components following Client-Server pattern. The application uses sockets to achieve proxy communication and each component can be run on separate machines. The idea of Auction House is fairly straight forward to understand. There are Auction House(s) where Agents come and bid on item(s). The Agent(s) and Auction House(s) must be registered with a Bank in order to keep track of funds. For more details on how to setup and usage, checkout the repo.

Mobile Agents

Mobile Agents is a program that manages the spread of fire on an internal graph consisting of nodes and agents protecting those nodes. Initally, the fire starts at a specific node (turns red) and then it spreads among neighboring nodes. The agent initially performs a random walk through the nodes until it finds a yellow node and clones itself. As the fire spreads, the agent clones itself on neighboring blue or yellow nodes to protect those nodes. The simulation process happens until the whole graph tree is burnt. For more details, please checkout the github repo.
